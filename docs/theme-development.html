Getting Started

Learn how to create beautiful, responsive themes for RailsPress from scratch.

Theme Structure

RailsPress themes follow a simple, organized structure that makes development intuitive and maintainable.

Directory Structure

your-theme/
├── config/
│   └── theme.json          # Theme metadata
├── views/
│   ├── layouts/
│   │   └── application.html.erb
│   ├── pages/
│   │   └── index.html.erb
│   └── shared/
│       ├── _header.html.erb
│       └── _footer.html.erb
├── assets/
│   ├── css/
│   │   └── theme.css
│   ├── js/
│   │   └── theme.js
│   └── images/
│       └── screenshot.png
└── helpers/
    └── theme_helper.rb

Theme Configuration

Every theme must include a theme.json file in the config directory: 

{
  "name": "Your Theme Name",
  "version": "1.0.0",
  "description": "A beautiful theme for RailsPress",
  "author": "Your Name",
  "license": "MIT",
  "tags": ["responsive", "modern", "blog"],
  "screenshot": "assets/images/screenshot.png"
}

Creating Your First Theme

1. Create Theme Directory

Create a new directory in app/themes/ with your theme name.

2. Add Configuration

Create the config/theme.json file with your theme metadata.

3. Create Layout

Build your main layout in views/layouts/application.html.erb.

4. Add Templates

Create page templates in the views/pages/ directory.

Liquid Template Engine

RailsPress uses Liquid for templating, providing a safe and flexible way to create dynamic content.

Basic Liquid Syntax

{{ page.title }}          
{% if user.logged_in %}     
  Welcome back!
{% endif %}

{% for post in posts %}     
  
{{ post.title }}

{% endfor %}

Available Objects

Site Data

  • site.name - Site title
  • site.description - Site description
  • site.url - Site URL

Page Data

  • page.title - Page title
  • page.content - Page content
  • page.excerpt - Page excerpt

User Data

  • user.name - User name
  • user.email - User email
  • user.avatar - User avatar

Posts Collection

  • posts - All posts
  • posts.latest - Latest posts
  • posts.featured - Featured posts

Styling Your Theme

RailsPress themes can use any CSS framework or custom styles. Popular choices include Tailwind CSS, Bootstrap, or custom CSS.

CSS Structure

/* assets/css/theme.css */
:root {
  --primary-color: #3b82f6;
  --secondary-color: #64748b;
  --background-color: #ffffff;
  --text-color: #1e293b;
}

body {
  font-family: 'Inter', sans-serif;
  background-color: var(--background-color);
  color: var(--text-color);
}

.header {
  background: linear-gradient(135deg, var(--primary-color), #8b5cf6);
}

Testing Your Theme

1. Local Development

Use the theme editor in the admin panel to test your changes in real-time.

2. Preview Mode

Preview your theme without activating it to ensure everything works correctly.

3. Responsive Testing

Test your theme across different devices and screen sizes using the builder's device preview.

Best Practices

Mobile-First Design

Always design for mobile devices first, then enhance for larger screens.

Semantic HTML

Use proper HTML semantics for better accessibility and SEO.

Performance Optimization

Optimize images, minify CSS/JS, and use efficient selectors.

Accessibility

Ensure your theme is accessible to users with disabilities.
